# Data dictionary

Schema reference for `runs_index.csv` and `all_metrics_long.parquet`.

## `runs_index.csv`

One row per run. Primary key: `run_slug` (also unique on `run_id`).
Generated by `scripts/refresh.py` from `runs_index.source.yaml` plus
wandb metadata. Hand edits are overwritten on next refresh — modify
`runs_index.source.yaml` instead.

| column | type | nullable | description |
|--------|------|:--------:|-------------|
| `run_slug` | str | no | `<dataset>_<variant>_<phase>_<run_id>` where phase ∈ {prefix, postfix}. Stable. |
| `run_id` | str | no | wandb 8-char ID. Immutable. |
| `display_name` | str | no | wandb display name. Mutable; do not use for joins. |
| `config_name` | str | no | hydra experiment config (e.g. `discrete_sbm_vignac_repro_exact`). |
| `wandb_project` | str | no | wandb project string under `graph_denoise_team`. |
| `wandb_url` | str | no | full URL. |
| `dataset` | enum | no | `sbm` \| `enzymes`. |
| `variant` | enum | no | `vignac`, `vignac_spectral`, `pearl`, `pearl_spectral`, `pearl_gnnconv_norm`, `pearl_gnnconv_raw`. |
| `postfix` | bool | no | True iff trained with the post-mask-fix architecture. |
| `parent_run_id` | str | yes | Run this one continues from (any kind of continuation). |
| `parent_run_slug` | str | yes | Same, as slug. Filled together with `parent_run_id`. |
| `continuation_type` | enum | no | See "Continuation types" below. |
| `launched_at_utc` | iso8601 | no | from wandb `createdAt`. |
| `ended_at_utc` | iso8601 | yes | null while running. |
| `final_state` | enum | no | `running`, `finished`, `crashed`, `failed`, `killed`, `cancelled`, `oom`, `preempted`. |
| `final_step` | int | no | last `trainer/global_step` (or `_step` fallback). |
| `health` | enum | no | `stable`, `elevated_grad`, `blew_up`, `invalidated_mask_bug`. |
| `notes` | str | yes | one-line free text. |

### Continuation types

| value | meaning |
|-------|---------|
| `original` | First run for this (dataset, variant, phase) cell. Parent fields null. |
| `auto_reassign_force_fresh` | Modal auto-reassigned the call; trainer relaunched with `force_fresh=True`. Started at step 0. |
| `auto_reassign_resume` | Modal auto-reassigned; trainer launched with `force_fresh=false`. Step counter continues. |
| `manual_resume_from_checkpoint` | Operator-initiated relaunch with `force_fresh=false +run_id=<…>`. Continues training; new wandb run. |
| `post_fix_replacement` | Architectural replacement after the mask-bug fix (logical pointer; technically a fresh run). |

### Health labels

| value | meaning |
|-------|---------|
| `stable` | grad_norm < 5, lr in expected range, no numerical issues at point of observation. |
| `elevated_grad` | grad_norm 1–5, training still converging but with margin to investigate. |
| `blew_up` | grad_norm ≫ 100 or lr → ∞. Numerical instability. |
| `invalidated_mask_bug` | Trained with the pre-fix mask divergence; metrics not byte-equivalent to upstream DiGress; not a usable reference for paper claims. |

## `all_metrics_long.parquet` (generated locally, not committed)

One row per `(run_slug, _step, metric_name)` triple. Long/tidy format,
parquet-encoded for size (the CSV equivalent is ~1.2 GB; parquet is
~10–20× smaller and pandas/polars/duckdb load it directly).

This file is **regenerated by `scripts/refresh.py`** and is **not
committed** to git (gitignored under
`.gitignore` for `paper-artifacts/repro-ablations/data/all_metrics_long.parquet`).
Run `doppler run -- uv run scripts/refresh.py` from the bundle root
to regenerate.

Use `pivot_table` to widen for plotting. Only scalar metrics are
included; image / wandb-media keys live under `media/per_run/<run_id>/`.

```python
import pandas as pd
df = pd.read_parquet("data/all_metrics_long.parquet")
```

| column | type | nullable | description |
|--------|------|:--------:|-------------|
| `run_slug` | str | no | foreign key into `runs_index.csv`. |
| `run_id` | str | no | denormalised from index (filterable without join). |
| `postfix` | bool | no | denormalised. |
| `dataset` | enum | no | denormalised. |
| `variant` | enum | no | denormalised. |
| `_step` | int | no | wandb internal step counter. |
| `trainer_global_step` | int | yes | training-loop step (when logged that row). |
| `epoch` | int | yes | filled when that row had `epoch` logged. |
| `timestamp_utc` | iso8601 | no | from wandb `_timestamp`. |
| `metric_namespace` | enum | no | namespace label (see below). |
| `metric_name` | str | no | full wandb key. |
| `metric_value` | float | no | scalar value. NaN rows are dropped at refresh time. |

### Metric namespaces

Derived from `metric_name` by `classify_metric_namespace()` in `refresh.py`.

| namespace | matches | examples |
|-----------|---------|----------|
| `mmd` | `gen-val/*` | `gen-val/degree_mmd`, `gen-val/clustering_mmd`, `gen-val/orbit_mmd`, `gen-val/spectral_mmd` |
| `train` | `train/*` | `train/loss_epoch`, `train/loss_step` |
| `val` | `val/*` | `val/epoch_NLL`, `val/loss` |
| `diagnostics` | `diagnostics-{train,val}/*` (excluding `progress/`) | `diagnostics-train/opt-health/grad_norm_total`, `diagnostics-train/opt-health/effective_lr` |
| `progress` | `diagnostics-{train,val}/progress/*` | `diagnostics-train/progress/loss_per_t/bin_0`, `diagnostics-val/progress/bits_per_edge` |
| `system` | `system/*` | `system/gpu.0.memoryAllocated`, `system/cpu`, `system/memory` |
| `impl_perf` | `impl-perf/*` | `impl-perf/train/step_time_s` |
| `lr` | `lr-*` and `train/lr` | `lr-AdamW`, `train/lr` |
| `other` | anything else | `epoch`, `_runtime`, `trainer/global_step` |

### Units caveat — MMD values are squared

The `mmd` namespace contains **squared MMD² values** (V-statistic biased
estimator). No square root before comparing to DiGress paper Table 1
ratios or HiGen Table 1 raw values. See `../context/mmd-units-and-protocol.md`.

## Per-run parquets

`per_run_history/<run_id>.parquet` is the wandb history exported
verbatim, wide format, one row per `_step`, scalar-only. Re-meltable
under different namespace rules without re-querying wandb.
